Application Container Hands-on Labs

Introduction

This is a series of 11 hands-on labs designed to familiarize you with the Application Container functionality of Oracle Multitenant. In these labs, we follow the journey of a notional company, Walts Malts, as their business expands from a single store to a global powerhouse from startup to starship.

 

These labs will be run in SQL Plus or SQLcl. The lab scripts are in your VM in folder HOL/wmLabs. Each lab in this series is run from a single script. These are intended to be run in sequence, as there are dependencies between the scripts. The eleven scripts are as follows:

Lab 1 01.Instant_SaaS.sql

Lab 2 02.Upgrade_v1_v2.sql

Lab 3 03.Containers_Queries.sql

Lab 4 04.Compatibility.sql

Lab 5 05.Expansion_ARRs.sql

Lab 6 06.Durable_Location_Transparency.sql

Lab 7 07.Data_Sharing.sql

Lab 8 08.Patch_301.sql

Lab 9 09.DBA_Views.sql

Lab 10 10.Diagnose_Restart.sql

Lab 11 11.Container_Map.sql

Format of Labs

These labs are in the "EasyLab" format, simulating what you might type in SQLPlus or SQLcl, but saving you the effort and distraction of transcribing from a workbook, so you can focus on what is happening. With EasyLab scripts, simply start the script, read what's happening and press [Enter] when ready to execute the next batch of commands. Ample comments are provided to explain what we're doing.

Initialization

The lab scripts use a small number of variables which must be correctly set according to your environment. These relate to things such as your Listener host name and path in which your data files will be created. These are defined in file profile.sql. Before you begin, you should open this in a text editor and verify that the settings are correct. The key variables to be set include the following:

 

Variable

Sample Value

Note

PDB_Dir

/home/oracle/oradata

The path in which your data files will be stored

hostname

localhost

The name of the host for your database server

PDB_Suffix

_07

A suffix to distinguish one participants work from anothers (in shared environments)

CDB1_Name

CDB1_07

We need two CDBs. This is the name of the first one.

CDB2_Name

CDB2_07

We need two CDBs. This is the name of the second one.

 

The settings for these variables should be changed if necessary before you proceed.

 

Before you begin, you need to set up the environment properly; with the CDBs properly initialized and the required database users in place. To do this, open a terminal window on your VM and issue the following commands:

 

cd /HOL/wmLabs

sqlplus /nolog

@Setup_Lab0

 

This will execute the scripts required to initialize the environment properly, including creating the administrative users.
